## 脆弱性の報告

私たちはセキュリティの脆弱性を深刻に受け止めています。セキュリティの問題を発見した場合は、責任を持って報告してください。

### 報告方法

**セキュリティの脆弱性をGitHubの公開Issueで報告しないでください。**

代わりに、以下のメールアドレスに送信してください：

**devtools-support@lotusandcompanyinc.com**

### 報告に含める内容

報告には以下の情報を含めてください：

1. **説明**: 脆弱性の明確な説明
2. **再現手順**: 問題を再現するための詳細な手順
3. **影響**: 脆弱性の潜在的な影響
4. **影響を受けるバージョン**: どのバージョンが影響を受けるか
5. **修正案**: 問題の修正方法に関する提案がある場合（任意）

### 対応タイムライン

- **初期対応**: 48時間以内
- **状況更新**: 7日以内
- **解決目標**: 30日以内（重大度に応じて変動）

### 対応の流れ

1. **確認**: 48時間以内に報告の受領を確認します
2. **調査**: 問題を調査し、その影響を判断します
3. **更新**: 進捗状況をお知らせします
4. **解決**: 修正後、公開前にお知らせします
5. **クレジット**: セキュリティアドバイザリーであなたの功績を認めます（匿名を希望する場合を除く）

## 公開ポリシー

- 私たちは協調的な公開プロセスに従います
- セキュリティの問題は修正が利用可能になった後に公開されます
- 適切な公開タイムラインを決定するために協力します
- 重大な脆弱性が活発に悪用されている場合は、より早く公開する場合があります

## セキュリティのベストプラクティス

DevToolsを使用する際：

1. **最新の状態を維持**: 常に最新バージョンを使用してください
2. **公式ソースからダウンロード**: 公式GitHubリポジトリからのみダウンロードしてください
3. **整合性の確認**: 利用可能な場合はファイルハッシュを確認してください
4. **問題を報告**: 疑わしい動作に気づいた場合は報告してください

## 適用範囲

このセキュリティポリシーは以下に適用されます：

- DevToolsアプリケーション
- GitHubの公式リリース
- このリポジトリ内のドキュメントと関連ファイル

## 適用範囲外

以下は一般的に適用範囲外です：

- サードパーティの依存関係の問題（それぞれのプロジェクトに報告してください）
- ソーシャルエンジニアリング攻撃
- 物理的なセキュリティの問題
- ユーザーのデバイスへの物理的なアクセスを必要とする問題
